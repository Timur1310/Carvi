# Автоматическое улучшение Carvi

Раз в неделю Claude Code сам проходит по сайту, чинит и улучшает его, оптимизирует под поиск, коммитит и пушит в GitHub — Vercel деплоит автоматически. Ничего вводить руками не нужно.

## Из чего состоит

- `improve-site.md` — инструкция (что именно делать каждый прогон). Её выполняет Claude Code.
- `run-weekly-improve.bat` — запускает Claude Code в фоновом режиме и пишет лог в `automation/logs/`.
- Планировщик задач Windows — дёргает `.bat` по расписанию.

## Разовая настройка

**1. Удали залипший git-файл** (остался от прошлой сессии, иначе git будет ругаться):

```
del C:\Users\Lenovo\Projects\myapp\.git\index.lock
```

**2. Проверь, что Claude Code установлен и залогинен.** В обычном терминале:

```
claude --version
```

Если команды нет — установи Claude Code и один раз войди в аккаунт (`claude`, затем следуй подсказкам). Автозадача использует уже сохранённую авторизацию.

**3. Проверь, что git пушит без пароля.** Один раз выполни в папке репозитория `git push` вручную — если просит логин, настрой GitHub-доступ (Personal Access Token или GitHub CLI `gh auth login`), чтобы автозапуск не завис на вводе пароля.

**4. Прогони вручную один раз** (важно — убедиться, что всё работает, прежде чем ставить на расписание):

```
C:\Users\Lenovo\Projects\myapp\automation\run-weekly-improve.bat
```

Посмотри свежий лог в `automation\logs\` и коммит в GitHub.

**5. Поставь на еженедельное расписание** (понедельник, 10:00). В терминале:

```
schtasks /create /tn "Carvi Weekly Improve" /tr "C:\Users\Lenovo\Projects\myapp\automation\run-weekly-improve.bat" /sc weekly /d MON /st 10:00 /f
```

Ежедневно вместо еженедельно — замени `/sc weekly /d MON` на `/sc daily`.

## Полезное

- Посмотреть задачу:  `schtasks /query /tn "Carvi Weekly Improve"`
- Запустить сейчас:   `schtasks /run /tn "Carvi Weekly Improve"`
- Удалить задачу:     `schtasks /delete /tn "Carvi Weekly Improve" /f`
- Логи каждого прогона лежат в `automation\logs\`.

## Важно

- Компьютер должен быть **включён** в момент запуска (10:00 понедельника). Задачу можно настроить «запускать при следующем включении, если пропущено» в графическом Планировщике задач (`taskschd.msc`).
- Каждый прогон делает **небольшие безопасные правки**, а не крупные переделки — так безопаснее. Если что-то не понравилось, откатывается обычным `git revert`.
