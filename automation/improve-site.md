## Правила
- **Маленькие, безопасные, обратимые изменения.** Никаких массовых переписываний за один прогон. Цель — стабильный прогресс, а не разовый рывок.
- **Ничего не удаляй оптом** (контент, страницы, данные). Улучшай и добавляй.
- Сохраняй существующий стиль кода, переносы строк (CRLF) и кодировку UTF-8. Не трогай файлы ради одних переносов строк.
- Русский язык — грамотный и естественный. Проверяй заголовки/описания на кривые формулировки.
- Не подключай внешние скрипты (на сайте строгий CSP). Аналитику/рекламу добавляй только если у пользователя есть ID — иначе оставь заметку в отчёте.

## Шаги каждого прогона

1. **Синхронизация.** В папке репозитория: `git pull --rebase origin main`. Если рабочее дерево грязное — сначала разберись (`git status`).

2. **Аудит.** Найди 3–7 конкретных проблем с наибольшей пользой для продвижения и качества. Смотри в первую очередь:
   - SEO: у каждой страницы уникальные `<title>`, `<meta name="description">` (≤160 симв.), `<link rel="canonical">`, OG/Twitter-теги, `<html lang="ru">`.
   - JSON-LD structured data на ключевых страницах (index, cars, tops, learn, quiz; у `vs/` уже есть).
   - Битые внутренние ссылки и ссылки на несуществующие файлы/картинки.
   - Картинки без `alt`, отсутствующие размеры, тяжёлые файлы.
   - `sitemap.xml`: все страницы включены, URL на `https://carvi.vercel.app`, нет мёртвых ссылок; `robots.txt` корректен.
   - Кривой/нечитаемый русский в заголовках, кнопках, описаниях.
   - Доступность (a11y) и мобильная вёрстка — очевидные проблемы.
   - Скорость: лишний инлайн, дубли CSS, не-lazy картинки.

3. **Правки.** Внеси выбранный набор улучшений. Держи диффы небольшими и осмысленными.

4. **Регенерация SEO-страниц.** Если менял `js/cars-data.js` — перегенерируй `vs/` и `sitemap.xml`:
   `node tools/gen-compare.js https://carvi.vercel.app`

5. **Проверка (обязательно перед коммитом).**
   - HTML не побит (теги закрыты, нет пустых `content=""` у важных мета).
   - JSON-LD валиден (`node -e` с `JSON.parse`).
   - `sitemap.xml` — валидный XML, число `<url>` не уменьшилось без причины.
   - Нет новых битых относительных ссылок.
   - `git diff --stat` показывает только осмысленные изменения (без паразитных переносов строк по всему репо).

6. **Changelog.** Добавь короткую запись в `CHANGELOG.md` (создай, если нет): дата + что улучшено и почему это помогает продвижению.

7. **Коммит и деплой.**
   - `git add -A`
   - `git commit -m "Weekly auto-improve: <краткое содержание>"`
   - `git push origin main`
   - Vercel задеплоит автоматически из GitHub.

8. **Итог.** Короткий отчёт: что изменил, что перегенерил, хеш коммита, 1–2 идеи для следующего прогона. Если ничего стоящего не нашёл — так и скажи и НЕ делай пустой коммит.
